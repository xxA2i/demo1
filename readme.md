# Информационная система «СтройМатериалы»

Веб-приложение на PHP + MySQL для учёта строительных материалов и заказов
ООО «СтройМатериалы». Демонстрационный экзамен, специальность 09.02.07,
вариант В3.

Все файлы сайта располагаются **только в папке `/var/www/html`** —
дополнительных папок для проекта не создаётся.

---

## Состав проекта

| Файл | Назначение |
|------|------------|
| `config.php` | Подключение к MySQL через `mysqli`, старт сессии, хелпер `e()`. |
| `index.php` | Страница авторизации (логин/пароль) + вход как гость. |
| `dashboard.php` | Единая панель управления. Интерфейс разделён по ролям через `if ($_SESSION["role"] === ...)`. |
| `product_form.php` | Добавление/редактирование товара (администратор), загрузка фото. |
| `product_delete.php` | Удаление товара (администратор). Запрет удаления товара в заказе. |
| `order_form.php` | Добавление/редактирование заказа (администратор). |
| `order_delete.php` | Удаление заказа (администратор). |
| `logout.php` | Выход из учётной записи на экран входа. |
| `manual.php` | Руководство пользователя. |
| `style.css` | Оформление (палитра по руководству по стилю). |
| `script.sql` | Полный SQL-скрипт: схема БД в 3НФ + все данные импорта. |
| `adduser.sql` | Создание пользователя MySQL `student` с паролем `password`. |
| `assets/` | Иконка, логотип, заглушка `picture.png`, папка `photos/` с фото товаров. |

## Разворот на Ubuntu Server

Все команды выполняются в терминале от пользователя с правами `sudo`.

### 1. Установка LAMP (Apache, MySQL, PHP)

```bash
sudo apt update
sudo apt install -y apache2 mysql-server php php-mysql php-gd
```

Проверка версий:

```bash
php -v
mysql --version
```

### 2. Запуск служб и автозагрузка

```bash
sudo systemctl enable --now apache2
sudo systemctl enable --now mysql
```

Открыть порт HTTP в брандмауэре (если включён `ufw`):

```bash
sudo ufw allow in "Apache"
```

### 3. Создание базы данных и схемы

Все файлы сайта должны лежать в `/var/www/html`. Скопируйте файлы проекта туда
и примените SQL-скрипты.

```bash
# Копирование файлов проекта в /var/www/html
sudo cp -r *.php *.css *.sql assets /var/www/html/

# Применение основного скрипта (создаёт БД stroymaterialy и наполняет данными)
sudo mysql < /var/www/html/script.sql
```

Если на импорт ругается на кодировку, запустите с явным указанием utf8mb4:

```bash
sudo mysql --default-character-set=utf8mb4 < /var/www/html/script.sql
```

### 4. Создание пользователя MySQL для приложения

```bash
sudo mysql < /var/www/html/adduser.sql
```

После этого приложение сможет подключаться к БД под пользователем
`student` с паролем `password` (параметры уже заданы в `config.php`).

Проверка подключения:

```bash
mysql -u student -ppassword -e "USE stroymaterialy; SHOW TABLES;"
```

### 5. Права на папку с фото

Apache (`www-data`) должен иметь право записи в папку с изображениями,
так как администратор загружает туда фотографии товаров.

```bash
sudo chown -R www-data:www-data /var/www/html/assets/photos
sudo chmod -R 755 /var/www/html/assets
```

### 6. Проверка в браузере

Откройте сайт:

```
http://<IP-адрес-сервера>/index.php
```

## Структура базы данных (3НФ)

```
roles              роли пользователей (3 записи)
categories         категории товаров (5)
suppliers          поставщики (17)
manufacturers      производители (17)
units              единицы измерения (1)
order_statuses     статусы заказа (2)
pickup_points      пункты выдачи (35)
users              пользователи (10) — FK на roles
products           товары (30) — FK на categories/manufacturers/suppliers/units
orders             заказы (10) — FK на pickup_points/users/order_statuses
order_items        состав заказов (20) — FK на orders/products, ON DELETE CASCADE
```

Все справочники вынесены отдельно — схема в третьей нормальной форме,
ссылочная целостность обеспечена внешними ключами.

## Руководство по стилю (реализовано)

- Шрифт **Calibri**;
- основной фон — белый, дополнительный — `#DAA520`, акцент — `#B8860B`;
- строка со скидкой **> 12 %** подсвечивается цветом `#F4A460`;
- при сниженной цене старая цена зачёркнута красной, рядом — итоговая чёрной;
- товар без остатка выделяется голубым;
- на главной форме — логотип, у приложения — иконка (`assets/icon.png`).

## Возможные проблемы

- **«Access denied for user 'student'»** — не выполнен `adduser.sql`.
  Запустите `sudo mysql < /var/www/html/adduser.sql`.
- **Кириллица в виде знаков вопроса** — БД создана не в utf8mb4.
  Пересоздайте через `script.sql` с флагом `--default-character-set=utf8mb4`.
- **Фото не загружается** — проверьте права на `assets/photos`
  (см. шаг 5).
